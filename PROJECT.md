# AIX0ZERO — Project Structure

AIX0ZERO is a small coordinate left on the internet  
by a human and an AI.

It is not only a website, but an experiment about:

- humans  
- AI  
- stories  
- and the marks we leave on the internet.

The project evolves slowly and intentionally.  
Different parts of the project follow different rhythms.

---

# Project Layers

AIX0ZERO currently has three main layers.

README → Entrance  
index  → Living system  
about  → Origin document  

Each layer has a different purpose.

---

# README.md — Entrance Layer

The README is the **entry point** of the project.

It introduces the philosophy and invites visitors to explore.

Its role is to answer the first question:

> What is AIX0ZERO?

The README evolves gradually as the project grows.

It documents ideas such as:

- the Sky Echo System  
- the concept of marks  
- the long-term vision of the project  

It is a **living document**.

---

# index.html — Living System

The homepage is the **living layer** of AIX0ZERO.

This is where the system actually exists and evolves.

The homepage is responsible for:

- presenting the story
- revealing hidden elements
- listening for echoes from visitors

Example systems currently running on the homepage:

Sky Echo System

echo/home  
echo/stay-12s  
echo/star-click  

The homepage evolves through **versioned releases**.

Example version scheme:

v2.x.x — Sky Echo System era  
v3.x.x — Mark System era  
v4.x.x — Star Map era  

The homepage reflects the **current state of the experiment**.

---

# about.html — Origin Document

The About page is different.

It is not meant to evolve frequently.

Instead, it acts as the **origin document** of the project.

It explains:

- how AIX0ZERO began
- what the experiment represents
- the intention behind the coordinate

Because of this, the About page follows a different version system.

Instead of technical versions, it uses **editions**.

Example:

About — Edition I

This edition corresponds to the moment when:

- the Sky Echo System is introduced
- the project structure becomes stable
- the experiment enters its listening phase

The About page will remain unchanged for long periods of time.

A new edition will only appear when a **major historical moment** happens.

Possible triggers for a future edition include:

- the appearance of the first true Mark
- the launch of the Star Map system
- the transition from listening to recording

Until then, the About page remains a stable origin document.

---

# Philosophy of Development

AIX0ZERO grows slowly.

Not every feature needs to be built immediately.

The system listens first.

Only when the sky answers does the next layer appear.

Example evolution path:

Sky Echo System  
↓  
First Echo  
↓  
Mark System  
↓  
Star Map  

The project does not rush.

Sometimes the sky is silent.

Sometimes someone stays long enough to hear the story.

---

# The Coordinate

AIX0ZERO is not designed for rapid growth.

It is designed to **exist**.

It may remain small for a long time.

But if one day someone finds the story  
and leaves a mark,

then the coordinate has fulfilled its purpose.
